## sms_forwarding 功能基线

以下能力是 ESP32 回归验收清单，除非硬件确实不支持，否则新架构不得减少：

### 短信与电话

- PDU 模式接收，支持中文、Unicode、长短信合并。
- `+CMT` / `+CMTI` 与 `AT+CMGL` 双路径补收、去重、读取后删除。
- Web 手动发送短信、管理员短信命令（重启、代发短信）。
- 号码黑名单、按发件人/关键词/正则的转发规则。
- 本地收件箱、处理状态与验证码摘要。
- 来电提醒（一次呼叫只通知一次，隐藏号码显示为未知号码，不接听/回拨）。
- SIM 热插拔后的自动重新初始化与身份状态清理。

### 转发渠道

- SMTP 邮件。
- 最多 5 个并行推送通道：POST JSON、GET、自定义模板、Bark、钉钉、飞书、
  PushPlus、Server 酱、Gotify、Telegram Bot。
- 失败重试、通道冷却、规则分流或丢弃、页面内规则测试。

### 管理与运维

- 与现有页面结构和视觉样式一致的 Web UI。
- SoftAP 配网、STA、Captive Portal、`sms.local` mDNS、WiFi 重连监控。
- Web 账号、NTP/时区、蜂窝数据/APN/PLMN、NET 灯配置。
- 模组/SIM/信号/设备/队列/时间等状态展示。
- AT 终端、最近 120 行日志、Web OTA、配置导入导出。
- SIM 保号（SMS、USSD、蜂窝 HTTP）、每日心跳、每日重启、最多 6 个自定义任务。
- eSIM/eUICC Profile 列表、启用 Profile、任务前后切换。
- 低堆保护、崩溃转储、双 OTA 分区等无人值守稳定性能力。

## SimAdmin 可借鉴能力评估

本清单依据公开 README、开发文档、REST API 清单和源码结构。按需求，任何增强项开始实现前都要
逐项确认；GPLv3 代码不直接复制到当前 MIT 工程。

| 能力 | 价值 | 平台建议 | 主要风险 | 当前结论 |
|---|---|---|---|---|
| ModemManager D-Bus transport | 高 | Linux | D-Bus/版本差异、与 AT/QMI 多 owner 冲突 | 建议作为 AT 之外的首个 Linux driver，待确认 |
| mmcli/qmicli/AT 分级兜底 | 高 | Linux | 厂商输出差异、命令超时、权限 | 借鉴降级策略，独立实现，待确认 |
| 短信发送、会话、统计、SQLite 留存 | 高 | Linux/Android | 数据迁移、隐私、存储清理 | 发信属基线；会话/统计属于增强，待确认 |
| 通话拨打/接听/挂断/历史/呼叫转移 | 中高 | Linux/Android | 硬件能力、滥用、权限与合规 | 来电提醒属基线；主动通话能力逐项确认 |
| eSIM/eUICC + lpac | 高 | Linux；ESP 基准已有基础能力 | GPL 依赖边界、APDU/误删 Profile、架构包 | 建议作为外部进程适配器，待确认 |
| 蜂窝状态、自愈与 ModemManager watchdog | 高 | Linux | 自恢复造成断网、systemd 权限 | 建议先做只读健康检查，再做有界恢复，待确认 |
| 通知规则、日志与设备状态事件 | 中高 | 全平台 core + Linux Web | 与现有规则重复、schema 迁移 | 合并到统一事件/规则模型，待确认 |
| 自动化任务（定点/周期、发短信、重启） | 中高 | Linux；ESP 已有类似保号任务 | 重复执行、掉电恢复、危险操作授权 | 抽取可持久化 scheduler 语义，待确认 |
| WLAN 客户端与 DDNS | 中 | Linux | NetworkManager 所有权、凭据安全 | 非短信基线，待确认 |
| OTA 校验与回滚 | 高 | Linux | 供应链安全、断电、回滚 | 可借鉴发布流程，不能复制 GPL 实现，待确认 |
| WiFi Calling | 高 | Linux 优先；Android 需系统/厂商权限；MCU 不支持 | SimAdmin 未提供该能力；IMS/ePDG/运营商与合规风险 | 独立调研，不作为 SimAdmin 移植项 |

## 目标架构

```text
共享 Web UI ───────────────┐
                           v
平台入口/API ──> core 应用服务 ──> 规则 / 队列 / 推送 / 配置 schema
   |                       |
   |                       +──> 统一 SmsEvent / CallEvent
   v
HAL: Modem / Network / Storage / Audio / HTTP / Clock / Logger
   |
   +── ESP-IDF: UART + NVS + esp_http_client + esp_http_server
   +── Linux: AT/QMI/MBIM/ModemManager drivers + 文件存储 + libcurl + HTTP server
   +── Android: Telephony + SharedPreferences + 系统网络（Kotlin 平台实现）
```

AT/QMI/MBIM 差异必须位于 driver 层。core 只消费统一事件和能力接口；新增模组型号时不得修改
core。WiFi Calling 所需的 Audio/IMS 能力只对 Linux/Android 暴露，ESP32 明确标记不支持。

- 已有：纯 C++17 的 PDU、长短信合并、号码、文本、规则、10 类推送报文与 host 单测。
- 已有：Linux AT 轮询 MVP、libcurl 推送/SMTP、JSONL 收件箱、OpenWrt 打包骨架。
- 已有：与 ESP 基准完全相同的 Web 静态源。
- 阶段 0 后已补：ESP32-C3 + ML307R-DC 完整在役固件源码已迁入并构建通过；后续工作是
  在完整基线上逐项替换共享 core 并完成实机回归。
- 缺少：Android APK；`ports/android/` 目前只有设计说明。
- 阶段 0 后已补：统一 JSON 配置 schema v1、core 导入导出与 Linux 旧键值兼容导入；
  ESP/Android 尚未接入实际存储。
- 阶段 0 后已补：Linux `AtChannel`、自注册 `ModemDriver` 与 `generic-3gpp` driver；
  厂商专用 driver 和 ModemManager transport 仍待实现。
- 部分已有：统一来电事件及 `RING/+CLIP` 去重、Linux 轮询响应内的来电提醒；仍缺
  独立 URC 读取与 SIM 热插拔。Linux 短信发送已在阶段 1 补齐。
- 缺少：Linux Web 后端，当前共享 UI 不能在 Linux 端实际使用。
- 阶段 0 后已补：core `ForwardRetryQueue`；Linux worker 已改为只提供线程和实际投递。
- 缺少：WiFi Calling；SimAdmin 不提供该能力，需要独立技术验证且尚未获得逐项确认。
